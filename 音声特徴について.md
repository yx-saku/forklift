学習ノートブックでは、音声は **0.2秒ごとの固定窓** に分割され、各窓を以下の特徴量に変換して学習しています。

## 音声の前処理

- WAVを読み込み
- モノラル化
- サンプリング周波数を `AUDIO_SR = 16000` Hz に統一
- `WINDOW_SEC = 0.2` 秒ごとに分割  
  → 1窓あたり約 `3200` samples

## 学習に使う音声特徴量

### 1. 音量・波形系特徴

| 特徴量 | 意味 |
|---|---|
| `audio_rms` | RMS音量。音の大きさ |
| `audio_energy` | 平均二乗エネルギー |
| `audio_peak` | 最大絶対振幅 |
| `audio_ptp` | peak-to-peak。最大値 - 最小値 |
| `audio_zcr` | Zero Crossing Rate。波形の符号反転頻度 |
| `audio_missing` | 音声欠損フラグ |

衝撃音や大きな異音は、主に `audio_rms`, `audio_energy`, `audio_peak`, `audio_ptp` に出やすいです。

### 2. 周波数系特徴

FFTを使って、各0.2秒窓の周波数分布を見ています。

| 特徴量 | 意味 |
|---|---|
| `audio_centroid` | 周波数重心。音の明るさ・高低の傾向 |
| `audio_bandwidth` | 周波数帯域の広がり |
| `audio_high_freq_energy` | 3000Hz以上の高周波成分比率 |

金属音・擦過音・鋭い衝撃音などは `audio_high_freq_energy` に出やすいです。

### 3. log-mel特徴

`N_MELS = 16` なので、16次元のlog-mel特徴を作っています。

```text
audio_mel_00
audio_mel_01
...
audio_mel_15
```

これは音を16個のメル周波数帯に分解し、それぞれの強さをdBスケールで表したものです。  
RMSなどの単純な音量特徴では表しきれない「音色」や「衝撃音らしさ」を見るための特徴です。

## モデルへの入り方

音声モデルでは、以下の2グループだけを使っています。

```python
"audio_basic": True
"audio_mel": True
```

つまり、動き特徴は入れずに、

```text
audio_rms
audio_energy
audio_peak
audio_ptp
audio_zcr
audio_centroid
audio_bandwidth
audio_high_freq_energy
audio_missing
audio_mel_00〜audio_mel_15
```

を使って `audio_anomaly_score` を学習しています。

学習時はこれらを、

1. 欠損値補完: `SimpleImputer(strategy="median")`
2. 標準化: `StandardScaler()`
3. `IsolationForest`

に通しています。

要するに、音声は **音量・波形変化・周波数分布・log-mel音色特徴** に分解され、それを正常データのみで学習して、正常音から外れた窓を `audio_anomaly_score` として出しています。
